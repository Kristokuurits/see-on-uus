UML diagrammid

Unified Modeling Language (UML) on diagramm, mis võimaldab teil süsteeme ja tarkvara visualiseerida.
See on tarkvaraarenduse üks populaarsemaid diagrammide koostamise vorme ja sai ISO standardiks 1997. aastal.
Sellest ajast alates on tehtud mõned uuendused, et lisada täiendavaid UML-diagrammitüüpe ja toetada uusi tehnoloogiaid programmeerimisel.

UML Use Case diagramm kirjeldavad süsteemi kõrgetasemelisi funktsioone ja ulatust.
Need diagrammid näitavad ka süsteemi ja selle osalejate vahelisi koostoimeid.

UML Diagrammitüüpid on:

Use case diagram
Use case diagramme luuakse, kui vaatate oma süsteemi või programmi nõudeid.
Need esindavad funktsioone või tunnuseid, osalejaid ja nende omavahelist seost (nende suhteid).

Class Diagrams
Class Diagrammis on programmeerimiskeeled mis põhinevad klassidel ja nendevahelistel suhetel.
UML-i klassidiagrammide tähistus töötati välja nende programmide visuaalseks esitamiseks.

Interaction overview diagrams
Interaction overview diagrams Need UML-diagrammid on kombinatsioon tegevusskeemist ja järjestusskeemidest, kus iga üksiktegevus on paigutatud oma raamidesse, nii on lihtsam programmeerida.

Communication diagrams
Communication diagrams need on sarnased use case diagramidele
Objektidevahelise suhtluse visualiseerimiseks kasutatakse aga sagedamini järjestusskeeme, kuna need on selgemad ja rikkalikuma tähistusega.

Timing diagrams
Timing diagrams on eritüüpi järjestusskeem, mis jälgib objektide käitumist aja jooksul.
Neid kasutatakse sagedamini manustatud süsteemide, näiteks pesumasina juhtimistarkvara programmeerimisel.

Object diagrams
Object diagrams need visualiseerivad abstraktsemaid class diagramms.
Aitavad tarkvaraarendajatel kontrollida, kas nende abstraktsed struktuurid praktikas töötavad.

Component diagrams
Component diagrams näitab, kuidas keeruka tarkvarasüsteemi erinevad komponendid on omavahel seotud ja suhtlevad liideste abil. Neid ei kasutata lihtsamate või sirgjooneliste süsteemide jaoks.

Deployment diagrams
Deployment diagrams on veel üks spetsiaalne diagramm suurte ja keeruliste süsteemide jaoks, kus tarkvara on juurutatud mitmesse süsteemi.

Package diagrams
Package diagrams näitavad teie süsteemi sõltuvusi ja neid kasutatakse peamiselt siis, kui teil on suur koodibaas.
Package võivad sisaldada use case diagramme, et illustreerida süsteemi funktsionaalsust ja visualiseerida tarkvarasüsteemi kihilist arhitektuuri.

Composite structure diagrams
Composite structure diagrams Kui teil on keeruline tarkvarasüsteem, millel on palju erinevaid kasutusjuhtumeid, visualiseerivad UML-i Composite structure diagrams, mis programmi käivitamisel tegelikult juhtub.
